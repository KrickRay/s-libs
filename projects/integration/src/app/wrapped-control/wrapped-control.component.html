This integration exists because I tried eliminating WrappedFormControlSuperclass in favor of having only WrappedControlSuperclass that defaulted control to a FormControl. And everything looked great and all the tests passed, until I tried to use it in a real app. Then suddenly it wouldn't compile because the typing of AbstractControl wasn't a good enough match for a [formControl] binding. So we now have this in the integration app, and build it during CI.
